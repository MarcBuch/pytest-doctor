# pytest-doctor: Scoring

See [ARCHITECTURE.md](./ARCHITECTURE.md) for pipeline design and [CONFIG.md](./CONFIG.md) for score configuration.

pytest-doctor computes a single **0-100 health score** from normalized diagnostics generated by reused tools.

## Score Labels

- **75-100**: Excellent
- **50-74**: Needs Work
- **<50**: Critical

## Core Formula

```text
score = clamp(100 - total_penalty, 0, 100)
```

Where:

```text
total_penalty = coverage_penalty + quality_penalty + gaps_penalty + dead_code_penalty + complexity_penalty
```

## Severity Weights (Base)

Default base penalties per diagnostic:

- `error`: 5.0
- `warning`: 2.0
- `info`: 0.5

## Category Multipliers (Default)

To make score impact reflect risk:

- `gap/*`: `1.2`
- `coverage/*`: `1.0`
- `quality/*`: `1.0`
- `dead-code/*`: `0.8`
- `complexity/*`: `0.8`

Final diagnostic penalty:

```text
diagnostic_penalty = severity_weight * category_multiplier
```

## Suggested Implementation

```python
SEVERITY_WEIGHTS = {
    "error": 5.0,
    "warning": 2.0,
    "info": 0.5,
}

CATEGORY_MULTIPLIERS = {
    "gap": 1.2,
    "coverage": 1.0,
    "quality": 1.0,
    "dead-code": 0.8,
    "complexity": 0.8,
}


def category_bucket(diag_type: str) -> str:
    # diag_type is expected: quality|gap|coverage|dead-code|complexity
    return diag_type


def penalty_for(diag) -> float:
    base = SEVERITY_WEIGHTS.get(diag.severity, 2.0)
    mult = CATEGORY_MULTIPLIERS.get(category_bucket(diag.type), 1.0)
    return base * mult


def calculate_score(diagnostics: list) -> dict:
    totals = {
        "coverage": 0.0,
        "quality": 0.0,
        "gaps": 0.0,
        "dead_code": 0.0,
        "complexity": 0.0,
    }

    for d in diagnostics:
        p = penalty_for(d)
        if d.type == "gap":
            totals["gaps"] += p
        elif d.type == "dead-code":
            totals["dead_code"] += p
        elif d.type == "complexity":
            totals["complexity"] += p
        elif d.type == "coverage":
            totals["coverage"] += p
        else:
            totals["quality"] += p

    total_penalty = sum(totals.values())
    value = max(0.0, min(100.0, 100.0 - total_penalty))

    if value >= 75:
        label = "Excellent"
    elif value >= 50:
        label = "Needs Work"
    else:
        label = "Critical"

    return {
        "value": round(value, 1),
        "label": label,
        "breakdown": {k: round(v, 1) for k, v in totals.items()},
    }
```

## Why This Model Fits Minimal Architecture

- Tool-agnostic: scoring does not depend on internal analyzer implementations
- Stable: new tools can be added by mapping to existing diagnostic types
- Transparent: users can see exactly which category lowers score most

## Example

Given diagnostics:

- 2 `gap` errors -> `2 * (5.0 * 1.2) = 12.0`
- 5 `quality` warnings -> `5 * (2.0 * 1.0) = 10.0`
- 3 `dead-code` warnings -> `3 * (2.0 * 0.8) = 4.8`
- 2 `complexity` infos -> `2 * (0.5 * 0.8) = 0.8`

Breakdown:

- `gaps`: 12.0
- `quality`: 10.0
- `dead_code`: 4.8
- `complexity`: 0.8
- `coverage`: 0.0

Total penalty: `27.6`

Final score: `72.4` -> `Needs Work`

## Configuration Hooks

Keep these knobs small and explicit:

```json
{
  "minimum_score": 75,
  "scoring": {
    "severity_weights": {
      "error": 5.0,
      "warning": 2.0,
      "info": 0.5
    },
    "category_multipliers": {
      "gap": 1.2,
      "coverage": 1.0,
      "quality": 1.0,
      "dead-code": 0.8,
      "complexity": 0.8
    }
  }
}
```

## CI Gate Behavior

- Exit `0` when `score >= minimum_score`
- Exit `1` when `score < minimum_score`
- Exit `2` on execution/config errors
